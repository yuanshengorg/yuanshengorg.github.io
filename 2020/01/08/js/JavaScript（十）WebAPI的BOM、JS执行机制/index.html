    BOM

    • BOM是什么：

      • BOM（Browser Object Model）即浏览器对象模型
      • 它提供了独立于内容而与浏览器窗口进行交互的对象
      • 其核心对象是 window
      • BOM 由一系列相关的对象构成，并且每个对象都提供了很多方法与属性

    • BOM的构成：

      • 比DOM大，包含DOM；
      • 顶级对象：window；
        • document；
        • location；
        • navigation；
        • screen；
        • history；

    window对象

    window对象事件

    • onload 事件
    • DOMContentLoaded 事件
    • onresize 调整窗口大小事件

    定时器

    window 对象给我们提供了 2 个非常好用的方法-定时器。

    • setTimeout( )：一次性定时器
    • clearTimeout( )：停止一次性定时器
    • setInterval( ) ：重复定时器
    • clearInterval( )：停止重复定时器

    setTimeout( )

    <script>
    // setTimeout 
    // 语法规范 window.setTimeout(调用函数, 延时时间);
    // 这个window在调用的时候可以省略
    // 1. 这个延时时间单位是毫秒 但是可以省略，如果省略默认的是0        
    setTimeout(function() {
        console.log('时间到了');
    }, 2000);

    // 2. 这个调用函数可以直接写函数 还可以写 函数名 还有一个写法 '函数名()'
    function callback() {
        console.log('爆炸了');
    }
    setTimeout(callback, 3000);
    // setTimeout('callback()', 3000); // 我们不提倡这个写法

    // 3. 页面中可能有很多的定时器，我们经常给定时器加标识符 （名字)
    var timer1 = setTimeout(callback, 3000);
    var timer2 = setTimeout(callback, 5000);
</script>

    例：5秒后关闭广告

    <img src="images/ad.jpg" alt="一个5秒后隐藏的广告" class="ad">
<script>
    var ad = document.querySelector('.ad');
    setTimeout(function() {
        ad.style.display = 'none';
    }, 5000);
</script>

    例：停止定时器

    <button>点击停止定时器</button>
<script>
    var btn = document.querySelector('button');
    var timer = setTimeout(function() {
        console.log('如果不在5s内点击btn，就会爆炸了');
    }, 5000);
    btn.addEventListener('click', function() {
        clearTimeout(timer);
    })
</script>

    setInterval( )

    • setTimeout 延时时间到了，就去调用这个回调函数，只调用一次 就结束了这个定时器
    • setInterval 每隔这个延时时间，就去调用这个回调函数，会调用很多次，重复调用这个函数
    // setInterval 
// 语法规范：  window.setInterval(调用函数, 延时时间);
setInterval(function() {
    console.log('继续输出');
}, 1000);

    例：定时器效果

    <style>
    div {
        margin: 200px;
    }
    span {
        display: inline-block;
        width: 40px;
        height: 40px;
        background-color: #333;
        font-size: 20px;
        color: #fff;
        text-align: center;
        line-height: 40px;
    }
</style>
<div>
    <span class="hour">00</span>
    <span class="minute">00</span>
    <span class="second">00</span>
</div>
<script>
    // 1. 获取元素 
    var hour = document.querySelector('.hour'); // 小时的黑色盒子
    var minute = document.querySelector('.minute'); // 分钟的黑色盒子
    var second = document.querySelector('.second'); // 秒数的黑色盒子
    var inputTime = +new Date('2020-1-10 23:00:00'); // 返回的是用户输入时间总的毫秒数
    // 如果不先调用，会隔1s才有显示，定时器间隔1s
    countDown(); // 我们先调用一次这个函数，防止第一次刷新页面有空白 
    // 2. 开启定时器
    setInterval(countDown, 1000);
    // 之前实现的函数
    function countDown() {
        var nowTime = +new Date(); // 返回的是当前时间总的毫秒数
        var times = (inputTime - nowTime) / 1000; // times是剩余时间总的秒数 
        var h = parseInt(times / 60 / 60 % 24); //时
        h = h < 10 ? '0' + h : h;
        // 把剩余的小时给 小时黑色盒子
        hour.innerHTML = h; 
        var m = parseInt(times / 60 % 60); // 分
        m = m < 10 ? '0' + m : m;
        minute.innerHTML = m;
        var s = parseInt(times % 60); // 当前的秒
        s = s < 10 ? '0' + s : s;
        second.innerHTML = s;
    }
</script>

    例：清除定时器

    <button class="begin">开启定时器</button>
<button class="stop">停止定时器</button>
<script>
    var begin = document.querySelector('.begin');        
    var timer = null; // 全局变量  null是一个空对象
    begin.addEventListener('click', function() {
        // 本函数定义的timer不能在下一个函数中调用
        timer = setInterval(function() {
            console.log('定时器执行中……');
        }, 1000);
    })
    // 清除定时器 clearInterval()
    var stop = document.querySelector('.stop');
    stop.addEventListener('click', function() {
        clearInterval(timer);
    })
</script>

    例：发送短信后禁用按钮

    手机号码： <input type="number"> <button>发送</button>
<script>
    // 按钮点击之后，会禁用 disabled 为true 
    // 同时按钮里面的内容会变化， 注意 button 里面的内容通过 innerHTML修改
    // 里面秒数是有变化的，因此需要用到定时器
    // 定义一个变量，在定时器里面，不断递减
    // 如果变量为0 说明到了时间，我们需要停止定时器，并且复原按钮初始状态
    var btn = document.querySelector('button');
    var time = 3; // 定义剩下的秒数
    btn.addEventListener('click', function() {
        // 点击按钮后，disabled=true 禁用了按钮
        this.disabled = true;
        var timer = setInterval(function() {
            if (time == 0) {
                // 清除定时器和复原按钮
                clearInterval(timer);
                btn.disabled = false;
                btn.innerHTML = '发送';
                // 需要下一次重新开始，重新定义time=3
                time = 3;
            } else {
                btn.innerHTML = '还剩下' + time + '秒';
                time--;
            }
        }, 1000);
    })
</script>

    this指向问题

    1. 全局作用域或者普通函数中this指向全局对象window（注意定时器里面的this指向window）
    2. 方法调用中谁调用this指向谁
    3. 构造函数中this指向构造函数的实例

    上一例：发送短信后禁用按钮， this.disabled 可以改成this，表示btn自己；定时器里的 btn.innerHTML = '发送' 就不能改成this，这时是在定时器内，this表示window；

    <script>
    // this 指向问题 一般情况下this的最终指向的是那个调用它的对象
    // 1. 全局作用域或者普通函数中this指向全局对象window（ 注意定时器里面的this指向window）
    // 01
    console.log(this);
    // 02
    function fn() {
        console.log(this);
    }
    window.fn();
    // 03
    window.setTimeout(function() {
        console.log(this);
    }, 1000);

    // 2. 方法调用中谁调用this指向谁
    // 04
    var o = {
        sayHi: function() {
            console.log(this); // this指向的是 o 这个对象
        }
    }
    o.sayHi();
    // 05
    var btn = document.querySelector('button');
    btn.onclick = function() {
        console.log(this); // this指向的是btn这个按钮对象
    }
    // 06
    btn.addEventListener('click', function() {
        console.log(this); // this指向的是btn这个按钮对象
    })

    // 3. 构造函数中this指向构造函数的实例
    // 07
    function Fun() {
        console.log(this); // this指向的是fun实例对象
    }
    var fun = new Fun();
</script>

    location对象

    • 获取/设置url的对象；

    • url：统一资源定位符：

      • protocol://host[:port]/path/[?query]#fragment
      • http://www.halfbamboo.com/index.html?name=yuan&age=18#link

    location对象的属性

    例：5S后跳转到首页

    location.href设置值的使用；

    <button>点击</button>
<div></div>
<script>
    var btn = document.querySelector('button');
    // 点击按钮直接跳转    
    btn.addEventListener('click', function() {
        // console.log(location.href);
        location.href = 'http://www.halfbamboo.com';
    })
    // 5S后跳转
    var div = document.querySelector('div');
    var timer = 5;
    setInterval(function() {
        if (timer == 0) {
            location.href = 'http://www.halfbamboo.com';
        } else {
            div.innerHTML = '您将在' + timer + '秒钟之后跳转到首页';
            timer--;
        }
    }, 1000);
</script>

    例：获取URL参数

    location.search获取参数的使用；

    <!-- login -->
<form action="index.html">
    用户名： <input type="text" name="uname">
    <input type="submit" value="登录">
</form>

<!-- index.html -->
<div></div>
<script>
    console.log(location.search); // ?uname=andy
    // 1.先去掉？  substr('起始的位置'，截取几个字符);
    var params = location.search.substr(1); // uname=andy
    console.log(params);
    // 2. 利用=把字符串分割为数组 split('=');
    var arr = params.split('=');

    var div = document.querySelector('div');
    // 3.把数据写入div中
    div.innerHTML = arr[1] + '欢迎您';
</script>

    navigator对象

    • navigator 对象包含有关浏览器的信息；
    • 最常用的是 userAgent，返回由客户机发送服务器的 user-agent 头部的值；
    • 下面前端代码可以判断用户那个终端打开页面，实现跳转：
    if((navigator.userAgent.match(/(phone|pad|pod|iPhone|iPod|ios|iPad|Android|Mobile|BlackBerry|IEMobile|MQQBrowser|JUC|Fennec|wOSBrowser|BrowserNG|WebOS|Symbian|Windows Phone)/i))) {
    window.location.href = "";     //手机
 } else {
    window.location.href = "";     //电脑
 }

    JS执行机制

    单线程

    • JS语言的一大特点就是单线程；
    • 单线程就意味着，所有任务需要排队，前一个任务结束，才会执行后一个任务；
    • 如果前一个任务耗时很长，后一个任务就不得不一直等着；
    • 为了解决这个问题，利用多核 CPU 的计算能力，HTML5 提出 Web Worker 标准，允许 JavaScript 脚本创建多个线程，但是子线程完全受主线程控制；
    • 于是，JS 中出现了同步任务和异步任务；
    // 第一个问题 1,2,3
console.log(1);
setTimeout(function() {
    console.log(3);
}, 1000);
console.log(2);

// 2. 第二个问题 1,2,3
console.log(1);
setTimeout(function() {
    console.log(3);
}, 0);
console.log(2);

// 3. 第三个问题 1,2,3,click
//       如果click先点击的话1,2,click,3
console.log(1);
document.onclick = function() {
    console.log('click');
}
console.log(2);
setTimeout(function() {
    console.log(3)
}, 3000)

    同步和异步

    • 同步：
      • 前一个任务结束后再执行后一个任务；
      • 程序的执行顺序与任务的排列顺序是一致的、同步的；
    • 异步：
      • 做一件会花很长时间的事情时，可以同时去处理其他事情；

    JS中的同步任务（synchronous）和异步任务（asynchronous）

    同步任务：

    • 在主线程上排队执行的任务，只有前一个任务执行完毕，才能执行后一个任务；
    • 同步任务都在主线程上执行，形成一个执行栈；

    异步任务：

    • 不进入主线程、而进入”任务队列”的任务，当主线程中的任务运行完了，才会从”任务队列”取出异步任务放入主线程执行。
    • JS中的异步都是通过回调函数实现的；
    • 一般异步任务有以下三种类型：
      • 普通事件：click、resize等；
      • 资源加载：load、error等；
      • 定时器：setInterval、setTimeout等；
    • 异步任务是添加到任务队列中；（任务队列也称为消息队列）

    JS执行机制/事件循环

    1. 先执行执行栈中的同步任务；
    2. 异步任务（回调函数）放入任务队列中；
    3. 当执行栈同步任务执行完毕：再按次序读取任务队列中的异步任务；
    4. 被读取的异步任务结束等待状态，进行执行栈，开始执行；
    5. 主线程不断重复获得任务、执行任务、再获取任务、再执行，这种机制被称为 事件循环 （event loop）；
